1. return TracingRequest.builder().
    .productType(4)

    .build();

   Пример раннего связывания во время написания кода.
   Используется как заглушка для одного эндпоинта.
2. public static final String SM_SECOND = "MSECOND";
   public static final String SM_THIRD = "MTHIRD";
   public static final String SM_FOURTH = "MFOURTH";

   Пример позднего связывания во время компиляции.
   Исходя от использования определенной переменной регулируется набор отдаваемых в ответе или сохраняемых при запросе данных.
   Эти поля меняются или дополняются очень редко, частоты внедрения релизов вполне хватает для таких доработок.
   Поэтому выбрано связывание на данном этапе.

3. boolean isSystemParamEnabled = applicationService.getSystemParam();

   Пример позднего связывания во время выполнения.
   На основе значения системного параметра (поля в БД), определяется ход выполнения программы (используется при внедрении нового функционала, который будет регулироваться).
   Переменной присваиваться значение для каждой обработки новой заявки.
   Был выбран этот способ связывания, для оперативного регулирования (перехода на старые "рельсы") хода течения программы.
